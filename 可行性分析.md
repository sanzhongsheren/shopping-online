# 商品网上交易系统 可行性分析报告
**文档版本**：V1.0  
**开发团队**：5人小组（1设计+1前端+2后端+1数据库）  
**适用课程**：软件工程实践项目

---

## 1 引言
### 1.1 项目标识
- 项目名称：商品网上交易系统
- 文档类型：可行性分析报告
- 开发团队：5人学生小组
- 运行环境：PC端浏览器（Chrome/Edge/Firefox）

### 1.2 项目背景
本项目为软件工程课程实践项目，针对线下商品交易信息不透明、流程不规范、效率低等问题，开发一套轻量化、易使用的网上交易系统。系统实现用户注册登录、商品发布与审核、在线下单、订单跟踪等功能，满足课程实践与学习目标。

### 1.3 项目目标
1. 完成**用户管理、商品管理、审核管理、购物下单、订单管理**五大核心模块。
2. 采用前后端分离架构，实现完整交易流程闭环。
3. 交付可运行系统 + 全套软件工程文档。
4. 符合课程评分标准，界面规范、功能完整、文档齐全。

---

## 2 团队分工（5人固定分工）
1. **软件设计与沟通协调（1人）**
   - 负责系统设计、UML建模、文档编写、进度统筹、前后端对接沟通。
2. **前端开发（1人）**
   - 页面设计实现、界面开发、交互逻辑、接口联调。
3. **后端开发（2人）**
   - 负责接口开发、业务逻辑实现、权限控制、功能联调。
4. **数据库设计与维护（1人）**
   - 数据库表设计、SQL编写、数据存储、备份与优化。

## 3 项目技术选型（固定方案）
### 3.1 软件设计
- 建模工具：**PlantUML**（画用例图、类图、时序图、架构图）

### 3.2 前端技术
- 设计工具：**Figma**（界面原型、高保真设计）
- 界面框架：**Bootstrap**（响应式布局、快速开发）
- 开发工具：VS Code
- 开发语言：HTML/CSS/JavaScript

### 3.3 后端技术
- 开发工具：**Visual Studio Code**
- 开发语言：**JavaScript**
- 运行环境：**Node.js v24.14.1**
- 开发框架：**Express**
- 功能：接口开发、业务处理、数据交互

### 3.4 数据库
- 数据库类型：**SQL关系型数据库**
- 负责内容：表结构设计、SQL语句编写、数据增删改查、事务处理

## 4 可行性分析前提
### 4.1 功能需求
1. 用户注册、登录、角色区分（买家/卖家/管理员）
2. 商品发布、编辑、查询、展示
3. 商品与用户信息审核
4. 购物车、下单、订单生成
5. 订单状态更新（发货/收货）
6. 基础数据管理与查询

### 4.2 非功能需求
1. 支持10人以上同时在线使用
2. 页面响应快、操作简单
3. 密码加密、基础安全校验
4. 界面美观、布局规范

### 4.3 项目限制
- 开发周期：课程学期内（2–3个月）
- 经费：0成本，全部使用免费开源工具
- 规模：课程演示级别，不支持高并发商用

---

## 5 经济可行性
1. **零成本开发**
   - 所有工具免费：PlantUML、Figma、VS Code、Node.js、Express、SQL数据库
2. **无硬件成本**
   - 使用学生个人电脑即可完成开发
3. **无部署费用**
   - 本地运行或学生免费云服务器即可
4. **效益明确**
   - 掌握前后端全栈开发能力
   - 完成课程项目，提升团队协作能力
   - 可作为个人作品集

**结论：经济完全可行。**

---

## 6 技术可行性
### 6.1 技术成熟度
- PlantUML：轻量画图，语法简单，适合快速出图
- Figma + Bootstrap：前端设计与实现简单高效
- Node.js + Express：入门快、接口开发效率高，适合学生项目
- SQL数据库：表结构清晰，学习成本低，适合交易系统

### 6.2 团队能力匹配
- 设计：负责UML、文档、沟通，符合软件工程流程
- 前端：一人独立完成页面开发，Bootstrap降低难度
- 后端：两人分工开发接口，Express易上手
- 数据库：一人完成表设计与SQL编写

### 6.3 技术风险与应对
- 风险1：前后端对接不顺畅
  应对：设计负责人统一接口规范，定期沟通
- 风险2：数据库设计不合理
  应对：先出设计文档，团队评审后再开发
- 风险3：环境配置问题
  应对：统一Node.js版本，共享环境配置文档

**结论：技术完全可行。**
